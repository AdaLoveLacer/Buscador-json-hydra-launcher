Estrutura de Pastas e Responsabilidades

app/
layout.tsx

- Layout global da aplicação (wrappers, providers).

page.tsx

- Página raiz que redireciona para localidade (ex.: `/pt-BR`).

[locale]/page.tsx

- Página principal da UI internacionalizada. Integra `SearchBar`, `AdvancedFilters`, `DownloadCard`.

loading.tsx

- Componente de loading usado nas rotas.

components/
search-bar.tsx

- Componente de input para busca com debounce.

advanced-filters.tsx

- Painel lateral com filtros de data, tamanho, modo de busca e ordenação.

download-card.tsx

- Renderiza cada item de download (título, data, tamanho, source).

sources-manager.tsx

- Gerenciamento de fontes (adicionar/remover), UI para carregar fontes externas.

file-uploader.tsx

- Upload de arquivos locais (fonte) e leitura.

search-stats.tsx

- Mostra números (total, filtrados) e query atual.

ui/

- Biblioteca interna de componentes UI (botões, inputs, dialogs, cards).

... (button.tsx, card.tsx, etc.)

lib/
search.ts

- Núcleo da busca: normalização, caching, processamento, filtragem e scoring.

types.ts

- Tipos compartilhados TypeScript e utilitários para sinônimos.

url-cache.ts / idb.ts

- Cache local e persistência (IndexedDB) para dados de URLs.

utils.ts

- Funções utilitárias (parseFileSize, formatação, etc.).

hooks/
use-data-state.ts / use-app-state.tsx

- Stores (zustand) para estado global (`gogData`, `savedSources`).

use-mobile.ts / use-mounted.tsx

- Helpers para responsive e detecção de mount.

use-toast.ts

- Notificações (Toasts).

dev/ & tools/
dev/generate_large_test.js

- Ferramenta para gerar grandes conjuntos de dados para teste de performance.

tools/recover.py, tools/salvage_sqlite.py

- Scripts utilitários para recuperação de DB e salvamento.

Server Files
server.py / server_api.py

- Backend (Flask) para reidratar fontes salvas (ex: `GET /list`).

requirements.txt

- Dependências Python para o server.

Observações Técnicas e Próximos Passos

Pontos de atenção

Núcleo de Busca:

O núcleo está em `lib/search.ts`. Alterações afetam performance e scoring. Logs de debug detalhados já existem.

Normalização:

Usa normalização Unicode e remoção de acentos. Atenção ao regex Unicode (`\p{L}`).

Cache:

Usa WeakMap e Map. Essencial manter chaves estáveis (`title:uploadDate`).

Sugestões de próximos passos

Diagrama Detalhado:

Gerar diagrama detalhado por módulo (ex.: `lib/search.ts` dividido por funções).

Fuzzy Matching:

Implementar fuzzy matching (Levenshtein) para capturar erros de digitação (ex: `skyim` → `skyrim`).

Mais Testes:

Adicionar mais testes unitários cobrindo expansão de sinônimos e casos com pontuação/acentuação.
